## Summary

Implement notification management in the admin dashboard + backend with 3 sources: manual admin notifications (dashboard form в†’ edge function в†’ FCM broadcast), automatic notifications on offer creation (DB trigger в†’ edge function в†’ FCM broadcast), and automatic notifications on order status change to "shipped" (DB trigger в†’ edge function в†’ FCM targeted to order owner). Includes database migrations (image_url column, type constraint, unique token constraint), Edge Function updates (payload validation, token fetch, send, cleanup), and Flutter UI updates (type field, image upload wiring).

## Phase 1: Design

### Data Model Changes

**notifications table**:
- ADD COLUMN `image_url text`
- ADD CONSTRAINT `notification_type_check CHECK (type IN ('general', 'offer', 'order'))`

**user_fcm_tokens table**:
- ADD CONSTRAINT `unique_token UNIQUE(token)`

### Contracts

**Edge Function: notification_handler**
- Input: `{ title, body, type, offer_id?, product_id?, order_id?, image_url?, user_ids? }`
- Output: `{ success: true, notification_ids: [...] }` or `{ success: false, error: "..." }`
- Behavior: validate в†’ fetch tokens в†’ save notifications в†’ send FCM в†’ cleanup invalid tokens
